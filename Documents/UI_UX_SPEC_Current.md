## 1. 개요

### 1.1 문서 목적
네이버 블로그 크롤러의 GUI 인터페이스에 대한 UI/UX 명세서입니다. 사용자 인터페이스의 구성, 디자인, 상호작용 방식을 기술합니다.

### 1.2 대상 사용자
- **주 사용자**: 블로그 콘텐츠를 수집하고 분석하려는 사용자
- **기술 수준**: 초보자부터 전문가까지
- **사용 목적**: 블로그 포스트 대량 수집

### 1.3 기술 스택
- **GUI 프레임워크**: Tkinter (Python 표준 라이브러리)
- **스타일**: 시스템 기본 스타일 (ttk 위젯 사용)
- **플랫폼**: Windows 10/11 (현재 버전)

## 2. 디자인 원칙

### 2.1 사용성 (Usability)
- **직관적 인터페이스**: 복잡한 설정 없이 바로 사용 가능
- **명확한 피드백**: 각 작업에 대한 즉각적이고 명확한 피드백
- **오류 방지**: 입력 검증 및 확인 다이얼로그로 실수 방지

### 2.2 효율성 (Efficiency)
- **빠른 작업 완료**: 최소한의 클릭으로 작업 수행
- **진행 상황 표시**: 크롤링 진행 상황을 실시간으로 확인
- **재개 기능**: 중단된 작업을 쉽게 재개

### 2.3 명확성 (Clarity)
- **명확한 레이블**: 모든 입력 필드와 버튼에 명확한 레이블
- **도움말 제공**: 필요한 경우 도움말 버튼으로 안내
- **상태 표시**: 현재 작업 상태를 상태 표시줄에 표시

### 2.4 일관성 (Consistency)
- **일관된 레이아웃**: 모든 화면에서 일관된 레이아웃 구조
- **표준 위젯**: Tkinter 표준 위젯 사용
- **일관된 동작**: 유사한 작업에 대해 일관된 동작

```

#### 3.1.2 구성 요소

**1. 메뉴 바**
- **파일 메뉴**: 종료
- **설정 메뉴**: 설정 열기
- **도움말 메뉴**: 사용 방법

**2. 입력 방법 선택 영역**
- **라디오 버튼**: 단일 입력 / 파일 업로드 선택
- **블로그 ID 입력 필드**: 텍스트 입력 (50자)
- **파일 경로 표시**: 읽기 전용 텍스트 필드
- **파일 찾기 버튼**: 파일 선택 대화상자 열기

**3. 재개 옵션 영역**
- **체크박스**: 재개 모드 활성화/비활성화
- **체크포인트 파일 경로 표시**: 읽기 전용 텍스트 필드
- **체크포인트 찾기 버튼**: 체크포인트 파일 선택 대화상자 열기

**4. 설정 요약 영역**
- **설정 정보 표시**: 현재 설정값을 텍스트로 표시
- **설정 변경 버튼**: 설정 대화상자 열기

**5. 버튼 영역**
- **크롤링 시작**: 입력 검증 후 크롤링 시작
- **설정**: 설정 대화상자 열기
- **도움말**: 도움말 메시지 표시
- **종료**: 애플리케이션 종료

**6. 상태 표시줄**
- **상태 메시지**: 현재 작업 상태 표시
- **기본 메시지**: "준비"

#### 3.1.3 크기 및 배치
- **윈도우 크기**: 900x700 (기본)
- **최소 크기**: 800x600
- **패딩**: 10px
- **컴포넌트 간 간격**: 10px

- **프로그레스 바 업데이트**: 실시간으로 진행률 업데이트

## 4. 사용자 흐름

### 4.1 일반적인 크롤링 흐름

```
[메인 화면]
    ↓
[입력 방법 선택]
    ├─ 단일 입력 → [블로그 ID 입력]
    └─ 파일 업로드 → [파일 선택]
        ↓
[크롤링 시작 버튼 클릭]
    ↓
[입력 검증]
    ├─ 실패 → [에러 메시지 표시] → [메인 화면]
    └─ 성공 → [진행 상황 화면]
        ↓
[크롤링 진행]
    ├─ 중단 버튼 클릭 → [확인 다이얼로그]
    │   ├─ 취소 → [진행 상황 화면]
    │   └─ 확인 → [중단 처리] → [결과 화면]
    │
    └─ 완료 → [결과 화면]
        ↓
[결과 확인]
    ├─ 폴더 열기 → [파일 탐색기]
    ├─ 다시 크롤링 → [메인 화면]
    └─ 종료 → [애플리케이션 종료]
```

### 4.2 재개 흐름

```
[메인 화면]
    ↓
[재개 옵션 체크]
    ↓
[체크포인트 파일 선택]
    ↓

## 5. 상호작용 디자인

### 5.1 입력 필드

#### 5.1.1 블로그 ID 입력 필드
- **상태**: 활성화 / 비활성화
- **활성화 조건**: "단일 블로그 ID 입력" 선택 시
- **비활성화 조건**: "파일 업로드" 선택 또는 "재개 옵션" 체크 시
- **입력 검증**: 실시간 검증 (유효하지 않은 문자 입력 시 피드백)
- **포커스**: Tab 키로 이동 가능

#### 5.1.2 파일 경로 표시 필드
- **상태**: 항상 읽기 전용
- **표시 내용**: 선택된 파일의 전체 경로
- **수정 불가**: 사용자가 직접 입력 불가

#### 5.1.3 체크포인트 파일 경로 표시 필드
- **상태**: 항상 읽기 전용
- **표시 내용**: 선택된 체크포인트 파일의 전체 경로
- **활성화 조건**: "재개 옵션" 체크 시

### 5.2 버튼

#### 5.2.1 크롤링 시작 버튼
- **상태**: 항상 활성화
- **동작**: 
  1. 입력값 검증
  2. 실패 시 에러 메시지 박스 표시
  3. 성공 시 진행 상황 화면으로 전환
- **키보드 단축키**: Enter (포커스가 있을 때)

#### 5.2.2 중단 버튼
- **상태**: 크롤링 진행 중에만 활성화
- **동작**:
  1. 확인 다이얼로그 표시
  2. 취소 시 대화상자 닫기
  3. 확인 시:
     - 중단 신호 전송
     - 현재까지 데이터 저장
     - 체크포인트 저장
     - 결과 화면 표시
- **확인 메시지**: "크롤링을 중단하시겠습니까?\n진행 중인 작업은 저장됩니다."

#### 5.2.3 파일 찾기 버튼
- **동작**: 파일 선택 대화상자 열기
- **지원 파일 형식**: *.txt, *.csv
- **기본 디렉토리**: 사용자 문서 폴더

#### 5.2.4 체크포인트 찾기 버튼
- **동작**: 체크포인트 파일 선택 대화상자 열기
- **지원 파일 형식**: *.json
- **기본 디렉토리**: 체크포인트 디렉토리

#### 5.2.5 폴더 열기 버튼
- **동작**: 결과 파일이 있는 폴더를 파일 탐색기로 열기
- **플랫폼**: Windows (explorer 명령 사용)

### 5.3 라디오 버튼 / 체크박스

#### 5.3.1 입력 방법 선택 (라디오 버튼)
- **옵션**: 
  - "단일 블로그 ID 입력"
  - "파일로 여러 블로그 ID 업로드"
- **동작**: 선택 시 관련 입력 필드 활성화/비활성화

#### 5.3.2 재개 옵션 (체크박스)
- **동작**: 
  - 체크 시: 블로그 ID 입력 필드 비활성화
  - 체크 해제 시: 블로그 ID 입력 필드 활성화 (입력 방법에 따라)

### 5.4 프로그레스 바

#### 5.4.1 전체 진행률 표시
- **범위**: 0~100%
- **업데이트**: 실시간으로 진행률 계산 후 업데이트
- **표시 형식**: 프로그레스 바 + 텍스트 ("XX% (완료/전체)")

#### 5.4.2 예상 완료 시간
- **계산 방법**: 평균 처리 시간 기반 추정
- **표시 형식**: "예상 완료 시간: 약 X분 남음"
- **업데이트**: 주기적으로 재계산

### 5.5 로그 영역

#### 5.5.1 로그 메시지 형식
```
[HH:MM:SS] 메시지 내용
```

#### 5.5.2 로그 레벨 표시
- **INFO**: 기본 색상 (검정)
- **WARNING**: 경고 색상 (주황)
- **ERROR**: 에러 색상 (빨강)

#### 5.5.3 자동 스크롤
- **동작**: 새 로그 메시지 추가 시 자동으로 맨 아래로 스크롤
- **조건**: 사용자가 수동으로 위로 스크롤하지 않은 경우

## 6. 시각적 디자인

### 6.1 색상 스킴

#### 6.1.1 기본 색상
- **배경**: 시스템 기본 배경색 (흰색/밝은 회색)
- **텍스트**: 시스템 기본 텍스트 색상 (검정)
- **프레임**: 시스템 기본 프레임 색상 (회색)

#### 6.1.2 상태 색상
- **정상 상태**: 기본 색상
- **경고**: 주황색 (#FFA500)
- **에러**: 빨간색 (#FF0000)
- **성공**: 초록색 (#00FF00)

#### 6.1.3 프로그레스 바
- **완료 영역**: 파란색 (시스템 기본)
- **미완료 영역**: 회색 (시스템 기본)

### 6.2 타이포그래피

#### 6.2.1 폰트
- **기본 폰트**: 시스템 기본 폰트 (맑은 고딕 / Segoe UI)
- **크기**: 시스템 기본 크기 (일반적으로 9~11pt)

#### 6.2.2 텍스트 스타일
- **일반 텍스트**: 기본 폰트
- **레이블**: 기본 폰트 (굵게)
- **에러 메시지**: 빨간색 텍스트
- **성공 메시지**: 초록색 텍스트 (선택적)

### 6.3 레이아웃

#### 6.3.1 간격
- **윈도우 패딩**: 10px
- **컴포넌트 간 간격**: 10px
- **프레임 내부 패딩**: 10px

#### 6.3.2 정렬
- **왼쪽 정렬**: 대부분의 텍스트와 입력 필드
- **오른쪽 정렬**: 오른쪽 버튼 (종료 등)
- **중앙 정렬**: 프로그레스 바 레이블

#### 6.3.3 그룹화
- **LabelFrame**: 관련된 요소들을 그룹화
- **제목**: 각 그룹에 명확한 제목

### 6.4 아이콘 및 기호

#### 6.4.1 상태 표시 기호
- **성공**: ✓ (체크 마크)
- **경고**: ⚠ (경고 표시)
- **에러**: ✗ (X 마크)

#### 6.4.2 버튼 레이블
- **텍스트 기반**: 아이콘 없이 텍스트만 사용
- **명확한 액션**: 버튼 이름으로 기능 명확히 표시

## 8. 에러 처리 및 피드백

### 8.1 입력 검증

#### 8.1.1 실시간 검증
- **블로그 ID**: 입력 중 실시간 형식 검증 (필요 시)
- **피드백**: 유효하지 않은 입력 시 시각적 피드백 (필요 시)

#### 8.1.2 제출 시 검증
- **크롤링 시작 버튼 클릭 시**: 모든 입력값 검증
- **검증 실패**: 에러 메시지 박스 표시
- **검증 성공**: 크롤링 시작

**에러 메시지 예시**:
```
┌─────────────┐
│  오류       │
├─────────────┤
│  블로그 ID를 │
│  입력해주세요│
│             │
│  [확인]     │
└─────────────┘
```

### 8.2 작업 피드백

#### 8.2.1 진행 상황 피드백
- **프로그레스 바**: 시각적 진행률 표시
- **로그 메시지**: 텍스트로 상세 진행 상황 표시
- **상태 표시줄**: 현재 작업 상태 요약

#### 8.2.2 완료 피드백
- **성공 완료**: 
  - 결과 화면 표시
  - "✓ 크롤링이 성공적으로 완료되었습니다!" 메시지
- **중단 완료**:
  - 결과 화면 표시
  - "⚠ 크롤링이 중단되었습니다." 메시지
  - 재개 안내

### 8.3 에러 처리

#### 8.3.1 네트워크 오류
- **표시 방법**: 로그 영역에 에러 메시지 표시
- **색상**: 빨간색으로 강조
- **동작**: 다음 블로그로 계속 진행 (가능한 경우)

#### 8.3.2 파싱 오류
- **표시 방법**: 로그 영역에 경고 메시지 표시
- **색상**: 주황색으로 표시
- **동작**: 부분 데이터라도 수집하여 계속 진행

#### 8.3.3 치명적 오류
- **표시 방법**: 에러 메시지 박스 표시
- **동작**: 
  - 현재까지 데이터 저장
  - 체크포인트 저장
  - 사용자에게 오류 보고

**에러 메시지 박스 예시**:
```
┌─────────────────────────┐
│  오류                   │
├─────────────────────────┤
│  크롤링 중 오류가       │
│  발생했습니다:          │
│                         │
│  블로그를 찾을 수       │
│  없습니다: koding2002   │
│                         │
│  [확인]                 │
└─────────────────────────┘
```
